# 5. Latency

**Latency** is the time it takes for data to travel from the client to the server and for the response to come back.

In simple terms, it is the **delay between sending a request and receiving a response**.

For example, when you click a button on a website:

1. The request travels from your device to the server.
2. The server processes the request.
3. The response travels back to your device.

The total delay you experience is called **latency**.

## What Causes Latency?

Latency can be affected by:

- **Physical distance** between the client and server.
- **Network delays** while data travels through the internet.
- **Server processing time**.
- **Database queries**.
- **Large amounts of data** being transferred.

## How Can We Reduce Latency?

Latency can be reduced using:

- **CDNs** to serve content closer to users.
- **Caching** to avoid repeated processing or database queries.
- **Servers in multiple regions** to reduce physical distance.
- **Optimized database queries**.
- **Efficient network communication**.

## Key Point

> **Latency is the delay between making a request and receiving a response. Lower latency means a faster and more responsive system.**
